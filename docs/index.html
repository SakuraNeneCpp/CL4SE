v1.0.0 · Windows / macOS / Linux

Caps Lockで ・変換確定 ・改行

CL4SE (Caps Lock for Safe Enter) は、Caps Lockキーの挙動を日本語IMEの変換確定で上書きする常駐ツールです。 変換確定のつもりでEnterを押し、そのままメッセージを送ってしまう事故を減らします。必要なら、非変換中のCaps LockをShift+Enterによる安全改行に割り当てられます（初期状態はOFF）。

Why CL4SE

誤送信を防ぎます

特に英語圏で開発されたアプリでは、しばしばメッセージの送信にEnterが割り当てられます。 しかし日本語ではIMEの変換確定にもEnterが割り当てられ、これが誤送信の原因になります。 そこでCL4SEは使われていないCaps LockにIME確定を任せることで、誤送信を防止します。

01

簡単無料インストール

インストーラをダウンロードするか, このページの下の方にあるコマンドをコピペするだけで導入できます。アンインストールのコマンドもあるので、不満があれば簡単にアンインストールできます。

02

安心安全設計

IMEがオンで、変換中だと判断できる場合だけ確定キーを注入し、不明な状態では何もしないフェイルセーフ設計です。任意設定のShift+Enterも、IME状態が判定済みで非変換中の場合だけ動作します。また、オープンソースなので不審なバイナリでないことが検証できます。

03

3つのOSで同じ使い心地

Windows・macOS・Linuxに対応。共通の判定ロジックで、環境が変わっても操作はCaps Lockひとつです。安心してタイピングのマッスルメモリーに組み込めます。

Fail-safe by design

フェイルセーフ設計

CL4SEはキーボード入力を扱うからこそ、動作範囲を限定し、不確実な状況では何もしない設計を最優先しています。

  • IMEオン + 変換中
    確定できると判断     COMMIT
  • IME判定済み + 非変換中
    設定有効時だけ安全改行     SHIFT+ENTER
  • IME状態不明
    設定にかかわらず誤注入を避ける     DO NOTHING
01 / PRIVACY

入力内容を収集しない

入力した文字列は保存・外部送信せず、ログにも残しません。判定に使うのは制御キーの種別やIME状態など、動作に必要な情報だけです。

02 / LIMITED SCOPE

操作するキーを限定

抑止するのは物理Caps Lockだけ。注入するキーも、変換確定、安全改行、本来のCaps Lockパススルーに必要なものだけに限定します。

03 / CLEANUP

停止時に元へ戻す

通常停止や終了シグナルでフック、リマップ、仮想入力デバイスをクリーンアップします。macOSではdoctorから残留リマップを復元できます。

Runtime

実行環境

GUIや常駐ランタイムを追加せず、各OS向けの単一バイナリとして動作します。

x86_64

Windows

  • 管理者権限不要
  • MS-IME / Google 日本語入力
  • ログイン時自動起動に対応
  • UACセキュアデスクトップは対象外
UNIVERSAL

macOS

  • Apple Silicon / Intel
  • 入力監視・アクセシビリティ許可が必要
  • 標準日本語IMなどに対応

x86_64

Linux

  • X11 / Wayland
  • fcitx5 / IBus
  • evdev + uinput
  • inputグループ・udev設定が必要

One-line setup

インストール

OSを選び、コマンドをターミナルにコピペするだけ。ダウンロード、診断、ログイン時の自動起動登録、今すぐの起動までまとめて完了します。

Windows x64

PowerShellで実行します。管理者として起動する必要はありません。

PowerShell
導入 : インストール + スタートアップ登録 + 起動 
$ErrorActionPreference="Stop"; $d="$env:LOCALAPPDATA\Programs\CL4SE"; $e="$d\cl4se.exe"; New-Item -ItemType Directory -Force $d | Out-Null; if (-not (Test-Path $e)) { Invoke-WebRequest "https://github.com/SakuraNeneCpp/CL4SE/releases/download/v1.0.0/cl4se-windows-x86_64.exe" -OutFile $e }; & $e doctor; if ($LASTEXITCODE -ne 0) { throw "CL4SE doctor failed" }; & $e install-autostart; if ($LASTEXITCODE -ne 0) { throw "CL4SE autostart setup failed" }; & $e start; if ($LASTEXITCODE -ne 0) { throw "CL4SE background start failed" }
バックグラウンド実行を停止 
& "$env:LOCALAPPDATA\Programs\CL4SE\cl4se.exe" stop
バックグラウンド実行を開始・再開 
& "$env:LOCALAPPDATA\Programs\CL4SE\cl4se.exe" start
非変換時（idle時）のCaps Lock動作 既定: OFF
現在の設定を表示 & "$env:LOCALAPPDATA\Programs\CL4SE\cl4se.exe" setting show
安全改行をONShift+Enterを注入 & "$env:LOCALAPPDATA\Programs\CL4SE\cl4se.exe" setting idle-action shift-enter
安全改行をOFF何も注入しない（既定） & "$env:LOCALAPPDATA\Programs\CL4SE\cl4se.exe" setting idle-action none
本来のCaps Lock動作Caps Lockをトグル & "$env:LOCALAPPDATA\Programs\CL4SE\cl4se.exe" setting idle-action capslock
アンインストール 
$d="$env:LOCALAPPDATA\Programs\CL4SE"; $e="$d\cl4se.exe"; if (Test-Path $e) { & $e stop; & $e uninstall-autostart }; Remove-Item -LiteralPath $e -ErrorAction SilentlyContinue; Remove-Item -LiteralPath $d -ErrorAction SilentlyContinue

macOS universal

Apple Silicon / Intel共通です。初回だけTCC権限の手動許可が必要です。

Terminal
導入 
d="$HOME/Applications/CL4SE"; e="$d/cl4se"; mkdir -p "$d" && { [ -x "$e" ] || { curl -fL "https://github.com/SakuraNeneCpp/CL4SE/releases/download/v1.0.0/cl4se-macos-universal" -o "$e" && chmod 755 "$e"; }; } && { "$e" doctor >/dev/null 2>&1 || { "$e" start || true; sleep 3; "$e" stop || true; open "x-apple.systempreferences:com.apple.preference.security?Privacy"; }; } && "$e" install-autostart && "$e" start
バックグラウンド実行を停止 
"$HOME/Applications/CL4SE/cl4se" stop
バックグラウンド実行を開始・再開 
"$HOME/Applications/CL4SE/cl4se" start
非変換時（idle時）のCaps Lock動作 既定: OFF
現在の設定を表示 "$HOME/Applications/CL4SE/cl4se" setting show
安全改行をONShift+Enterを注入 "$HOME/Applications/CL4SE/cl4se" setting idle-action shift-enter
安全改行をOFF何も注入しない（既定） "$HOME/Applications/CL4SE/cl4se" setting idle-action none

アンインストール 
e="$HOME/Applications/CL4SE/cl4se"; [ ! -x "$e" ] || { "$e" stop; "$e" uninstall-autostart; }; rm -f "$e"; rmdir "$HOME/Applications/CL4SE" 2>/dev/null || true

システム設定が開いた場合だけ、「入力監視」と「アクセシビリティ」でCL4SEをオンにしてください。この確認だけはmacOSが手動操作を必須にしています。登録後は自動で起動します。

Linux x64

Wayland / X11、fcitx5 / IBusに対応します。最初にdoctorの案内で権限を整えます。

Shell
導入 
d="$HOME/.local/bin"; e="$d/cl4se"; s="$HOME/.local/share/cl4se"; r="/etc/udev/rules.d/99-cl4se-uinput.rules"; mkdir -p "$d" "$s" && { [ -x "$e" ] || { curl -fL "https://github.com/SakuraNeneCpp/CL4SE/releases/download/v1.0.0/cl4se-linux-x86_64" -o "$e" && chmod 755 "$e"; }; } && { getent group input >/dev/null || sudo groupadd input; } && { id -nG "$USER" | tr ' ' '\n' | grep -qx input || { sudo usermod -aG input "$USER" && : >"$s/managed-input-group"; }; } && { [ -e /dev/uinput ] || sudo modprobe uinput; } && { [ -f "$r" ] || { printf '%s\n' 'KERNEL=="uinput", SUBSYSTEM=="misc", GROUP="input", MODE="0660", OPTIONS+="static_node=uinput"' | sudo tee "$r" >/dev/null && : >"$s/managed-uinput-rule"; }; } && sudo udevadm control --reload-rules && sudo udevadm trigger --name-match=uinput && "$e" install-autostart && { systemctl --user stop cl4se.service >/dev/null 2>&1 || true; } && if sg input -c "'$e' doctor"; then { pgrep -x cl4se >/dev/null || sg input -c "'$e' start"; }; else "$e" uninstall-autostart; false; fi
バックグラウンド実行を停止 
"$HOME/.local/bin/cl4se" stop
バックグラウンド実行を開始・再開 
"$HOME/.local/bin/cl4se" start
非変換時（idle時）のCaps Lock動作 既定: OFF
現在の設定を表示 "$HOME/.local/bin/cl4se" setting show
安全改行をONShift+Enterを注入 "$HOME/.local/bin/cl4se" setting idle-action shift-enter
安全改行をOFF何も注入しない（既定） "$HOME/.local/bin/cl4se" setting idle-action none

アンインストール 
e="$HOME/.local/bin/cl4se"; s="$HOME/.local/share/cl4se"; r="/etc/udev/rules.d/99-cl4se-uinput.rules"; [ ! -x "$e" ] || { "$e" stop; "$e" uninstall-autostart; }; rm -f "$e"; if [ -f "$s/managed-uinput-rule" ]; then sudo rm -f "$r" && sudo udevadm control --reload-rules && sudo udevadm trigger --name-match=uinput; rm -f "$s/managed-uinput-rule"; fi; if [ -f "$s/managed-input-group" ]; then sudo gpasswd -d "$USER" input >/dev/null 2>&1 || true; rm -f "$s/managed-input-group"; fi; rmdir "$s" 2>/dev/null || true

途中でsudoパスワードを求められたら入力してください。inputグループとuinputルールも自動設定し、現在はすぐ起動、次回ログイン以降は自動起動します。アンインストール時は、この一行が追加した権限設定も戻します。

コマンドはv1.0.0固定です。更新時はReleaseページで最新版を確認してください。

ReleaseとSHA256SUMSを見る ↗

idle時の動作は、各OS欄の設定コマンドから変更できます。Shift+Enterによる安全改行は初期状態ではOFFです。実行中に変更するとバックグラウンド状態のまま安全に再初期化します。IME状態が不明な場合は設定にかかわらず何も注入しません。利用するチャットアプリでShift+Enterが「送信せず改行」になることを確認してから有効にしてください。
